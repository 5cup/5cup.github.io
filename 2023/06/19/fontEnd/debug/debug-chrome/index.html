前端调试--谷歌篇

发表于2023-06-19更新于2023-07-02

打开 Dev Tool

  • 菜单>更多工具>开发者工具
  • 快捷键 F12

打开命令菜单

  • 快捷键：

    • CTRL + SHIFT + P
    • Command + SHIFT + P(MAC)

  • 修改 DevTool 的主题
    在命令菜单中输入 dark/light theme 即可切换黑/白色主题

  • 截屏
    在命令菜单中输入 screenshot,会出现四个选项：

    • Capture area screenshot – 通过鼠标指针选中区域后会下载到电脑上
    • Capture full size screenshot 截取整个页面（包含滚动条之外的部分）并下载
    • Capture node screenshot 选中某个 node 节点后再执行该命令可将该节点进行截图并下载
    • Capture screenshot 截取当前显示页面（不包含滚动条之外的部分）并下载

  • dock 改变 DevTool 的位置

    • dock to left 将DevTool移至左侧
    • dock to right 将DevTool移至右侧
    • dock to bottom 将DevTool移至底部
    • undock 在新窗口打开DevTool

常用的 Tab

  • Element
  • Console
  • Source
  • Network
  • Application

Element

查找网页源代码 HTML 中的任一元素,手动修改任一元素的属性和样式且能实时在浏览器里面得到反馈

检查元素
可通过右键某个元素然后选择检查定位到该元素，也可点击 DevTool 左侧的图标来选中某元素,旁边的图标可切换成移动设备显示

查找 DOM 树

  • 快捷键

    • CTRL+F
    • Command+F
      通过快捷键可打开一个搜索框，在里面输入想要查询的内容即可，匹配的内容会被高亮显示

  • 查询方式

    • 文本查询 – 匹配符合的文本内容

    • css 选择器 – 通过 css 选择器指定，如输入 .zooming 来匹配 class 为 zooming 的元素

    • Xpath – //ul/li ‘//‘ 表示在全局范围内查找 ul 标签下面的 li 标签

  • Console
    也可以通过 Console 定位到某个节点
    在 console 中输入 inspect(document.getElementById('打开命令菜单'))回车，就能定位到该节点了

编辑样式

在 Element 面板的右半部分是样式内容，可直接在此修改页面的样式

style 面板

  • 让 hover 常驻
    选中某个元素后点击 :hov后能看到下方出现了多种状态，想要常驻那种状态直接勾选即可

  • 编辑 class
    选中某个元素后能看到该元素的 class，取消掉前面的勾选即可使该样式失效

    但是此种修改方法会应用到所有拥有该 class 的元素，若想单独修改某个元素的 class，可点击 .cls，，此时能看到有一个单独的 class 出现了，取消勾选只会对选中的元素生效

  • 复制样式
    选中某个元素后，右键选择 Copy> Copy Style 即可复制该样式

Computed 面板
Computed 将所有生效的样式都列举出来了，点击右边的箭头可跳转到包含该样式的 class 中

最底部是字体

Layout 面板
Layout 面板分为两个部分：Grid 和 FlexBox,页面中所用到的 grid 布局和 flex 布局都可以在此找到

Event Listeners 面板
此处可以看到页面中绑定的事件

DOM Breakpoints 面板
选中某个节点后右键选择 Break on，能看到三个选项

  • subtree modifications: 若此节点的子节点被修改了，js 运行就会被暂停
  • attribute modifications： 若节点的属性被修改了，js 运行就会被暂停
  • node removal： 若此节点被从文档删除，js 运行就会被暂停

Properties 面板
能看到所有节点属性

Accessibility 面板
帮助构建无障碍页面

Console（控制台）

  • 快捷键

    • CTRL + SHIFT + J
    • Command + Opton + J(mac)

  • 执行语句
    > 表示执行的语句
    <· 表示上一条语句的返回值

  • $_ 返回上一条语句的执行结果
    定义 a=1,b=2,执行 a+b 后，可使用$\_ 返回上一条语句(a+b)的结果

  • $0 返回当前选择的 DOM 节点
    在 选中一个元素节点后，在控制台输入 $0

    $1 返回上一个选择的节点，$2 返回上上一个选择的节点。。。

  • console 语句

    • console.log() 显示相关信息
    • console.error() 显示错误信息
    • console.warn() 显示警告信息
    • console.clear() 清空控制台信息
    • console.group() 将信息以组的形式展示
      组以 console.group()开始，以 console.groupEnd()结束
    • console.time() 返回一段代码执行的时间
      以 console.time()开始，以 console.timeEnd()结束
    • console.table() 把一个数组以表格的形式展示出来

  • Log 级别筛选
    可通过右边的下拉选项选择要显示的 Log 信息

  • 观测变量
    点击 眼睛图标可以观测变量变化
    定义变量 a = 3，点击眼睛图标后输入变量 a，再修改变量 a 的值，能从上方看到变量 a 的变化

Javascript 调试

可通过 debugger对代码进行调试
当执行到debugger语句时，页面会暂停

也可通过点击左边的行号来添加断点，添加后行号会变成蓝色

监测变量
可在右边的 Watch 中添加变量并对其进行监测

Event Listener Breakpoints
能够对特定事件进行监听，一旦检测到到该事件就会停止 js 运行
如下图一旦触发鼠标点击事件就就会停止 js 运行

Call Stack
在使用了框架（Vue）的情况下，若是依然选择 Event Listener Breakpoints 来检测某些事件，回直接跳转到 vue.js 源码里，此时可在 Call Stack 里右击选择 Add script to ignore list来忽略 vue.js 源文件，而是在自己编写的代码里暂停。

Network

字段介绍：

  • Name 请求的名称
  • Status 服务器返回的状态码
  • Type 请求的类型
  • Initiator 什么发起的请求
  • Size 资源的大小
  • Time 发起请求的时间
  • WateFall 瀑布流信息，可以看出时间具体花在了哪些部分。

不再监听网络请求
点击左上角的红色图标，待其变成灰色后表示已不再监听网络请求了，点击右边的图标可以清空所有请求

Preserve log
勾选 Preserve log 能保存历史请求信息，防止页面跳转后原来的请求信息的没了
Disable cache
去掉浏览器缓存
No throtting
节流器，能模拟用户的网络状态
Network conditions
能进行更多的网络设置

Import/Export HAR file
Export HAR file 将浏览器与服务器交互的数据以 JSON 格式保存起来
Import HAR file 导入别人的 HAR file 文件，可以用来分析别人的请求情况

Application

可以查看存在于客户端的一些数据，如 Local Storage、Session Storage、Cookies 等
