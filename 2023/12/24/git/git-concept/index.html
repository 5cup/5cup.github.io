Git 相关概念

发表于 2023-12-24 更新于 2023-12-26

仓库（Repository）

存储项目文件和历史记录的地方

克隆（Clone）

创建一个本地仓库的完整副本

提交（Commit）

保存对仓库中文件的更改

分支（Branch）

项目的不同版本线

合并（Merge）

将一个分支的更改合并到另一个分支

冲突（Conflict）

当合并两个分支时，相同文件的不同更改导致的问题

拉取请求（PullRequest）

在代码托管平台（如 GitHub）上请求将你的更改合并到原始仓库

推送（Pudh）

将本地更改发送到远程仓库

拉取（Pull）

从远程仓库获取更新并合并到本地仓库

暂存区（Staging Area）

临时保存改动的地方，在提交前

检出（Check out）

切换到不同的分支或提交

远程（Remote）

指向远程仓库的引用

标签（Tag）

标记特定的提交，通常用于版本发布

HEAD

指向当前分支最近一次提交的指针

rebase

一种合并的替代方法，用于重新排列提交的的顺序

gitignore 文件

定义 Git 应忽略的文件和目录的文件

分叉（Fork）

在代码托管平台上创建项目副本

索引（Index）

Git 内部的一个构成区域，保存了下一次提交将包含的文件列表

快照（Snapshot）

提交时，Git 对文件的状态进行快照，而不仅仅是记录差异

钩子（Hooks）

脚本，可以在特定动作发生时自动运行，如在提交前运行测试
